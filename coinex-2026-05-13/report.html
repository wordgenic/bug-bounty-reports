📋 Executive Summary

Comprehensive security assessment of CoinEx (coinex.com) conducted. The platform demonstrates strong security posture overall. One low-severity finding was identified but is explicitly listed as out-of-scope by the bug bounty program.

🎯 Finding: Open Redirect in /redirect Endpoint

Description

The CoinEx website contains an open redirect at /redirect?url= that redirects users to arbitrary URLs.

URL: https://www.coinex.com/redirect?url=https://evil.com
Method: GET
Parameter: url

Testing Results

PayloadResult
/redirect?url=https://evil.com302 redirect to /en?url=https://evil.com
/redirect?url=//google.comRelative redirect (stays on domain)

Impact

Explicitly listed as out-of-scope by CoinEx bug bounty program. Limited impact due to relative redirect behavior.

✅ Security Strengths Observed

  • Content Security Policy: Strong CSP headers in place
  • Cloudflare Protection: DDoS protection and WAF active
  • Security Headers: X-Frame-Options, X-XSS-Protection, HSTS all present
  • API Security: Public endpoints properly protected
  • No Obvious XSS: Reflected XSS testing returned no results
  • No SQL Injection: Public API parameters properly sanitized
