📋 Executive Summary

During an authorized bug bounty investigation of CoinEx (coinex.com, api.coinex.com), an open redirect vulnerability was identified in the /redirect endpoint. This finding is documented per program scope but falls under limited impact per the program's out-of-scope criteria.

🎯 Finding: Open Redirect in /redirect Endpoint

1. Description

The CoinEx website contains an open redirect endpoint at /redirect?url= that redirects users to arbitrary URLs. While the redirect is relative and limited to the coinex.com domain, it can be manipulated for phishing and social engineering attacks.

URL: https://www.coinex.com/redirect?url=https://evil.com
Method: GET
Parameter: url
Response: HTTP 302 → location: /en?url=https://evil.com

2. Testing Results

Test PayloadResponseStatus
/redirect?url=https://evil.com302 → /en?url=https://evil.comRedirects
/redirect?url=//google.com302 → /en?url=//google.comRelative redirect
/redirect?url=/admin302 → /en?url=/adminLocal redirect

3. Impact

Limited Impact (per program scope):

  • CoinEx explicitly lists open redirects as out of scope
  • Redirect is relative (stays within coinex.com domain)
  • Requires user interaction for phishing
  • Cannot be chained for credential theft due to relative nature

4. Why This Matters

Even though marked out-of-scope, open redirects can be combined with other vulnerabilities in sophisticated attack chains and are documented for completeness of the security assessment.
